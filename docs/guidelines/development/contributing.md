## 行動規範

このプロジェクトでは、オープンで友好的な環境を維持するために、以下の行動規範を設けています：

- 互いを尊重する
- 建設的なフィードバックを行う
- 多様性を受け入れる
- プロフェッショナルな態度を保つ

## 始め方

1. リポジトリをフォーク
2. 開発環境のセットアップ
   ```bash
   git clone https://github.com/YOUR-USERNAME/isebushi_karaoke.git
   cd isebushi_karaoke
   flutter pub get
   ```
3. 開発ブランチの作成
   ```bash
   git checkout -b feature/your-feature-name
   ```

## 開発フロー

1. **Issue の作成**
   - 新機能の提案
   - バグ報告
   - 改善提案

2. **ブランチ命名規則**
   ```
   feature/add-new-feature
   fix/bug-description
   improvement/enhancement-description
   ```

3. **開発プロセス**
   - [開発プロセスガイドライン](./process.md)に従う
   - タスクリストの作成
   - UMLの更新
   - 承認プロセスの遵守

## プルリクエスト

### PRの作成

1. **タイトル形式**
   ```
   feat: 新機能の追加
   fix: バグの修正
   improvement: 既存機能の改善
   refactor: コードのリファクタリング
   docs: ドキュメントの更新
   ```

2. **説明内容**
   ```markdown
   ## 変更内容
   - 具体的な変更点を箇条書きで
   
   ## 関連Issue
   - #Issue番号
   
   ## スクリーンショット（UI変更時）
   
   ## テスト結果
   - 実施したテストの概要
   - テストの結果
   
   ## チェックリスト
   - [ ] テストを追加/更新した
   - [ ] ドキュメントを更新した
   - [ ] コーディング規約に従っている
   ```

### レビュープロセス

1. **レビュー基準**
   - コーディング規約の遵守
   - テストの網羅性
   - パフォーマンスへの影響
   - セキュリティ考慮

2. **レビューフロー**
   - 最低1名のレビュアーの承認が必要
   - CI/CDパイプラインの成功確認
   - コードオーナーの最終承認

## コーディング規約

1. **Dart/Flutter規約**
   - [Effective Dart](https://dart.dev/guides/language/effective-dart)に準拠
   - [Flutter Style Guide](https://github.com/flutter/flutter/wiki/Style-guide-for-Flutter-repo)に従う

2. **プロジェクト固有の規約**
   - ファイル名: スネークケース
   - クラス名: アッパーキャメルケース
   - 変数/メソッド名: ローワーキャメルケース
   - 定数: アッパースネークケース

3. **コメント規則**
   ```dart
   /// ドキュメンテーションコメント
   /// - パラメータの説明
   /// - 戻り値の説明
   /// - 使用例
   
   // 実装の説明や注意点
   ```

## コミットメッセージ

1. **形式**
   ```
   <type>: <subject>
   
   <body>
   
   <footer>
   ```

2. **Type**
   - feat: 新機能
   - fix: バグ修正
   - docs: ドキュメント
   - style: フォーマット
   - refactor: リファクタリング
   - test: テスト
   - chore: その他

3. **例**
   ```
   feat: ピッチ検出アルゴリズムの改善
   
   - FFTベースの新アルゴリズムを実装
   - ノイズ除去処理を追加
   - 処理速度を20%改善
   
   Closes #123
   ```

## テスト

1. **必須テスト**
   - 単体テスト
   - ウィジェットテスト
   - 統合テスト

2. **テストカバレッジ**
   - 最低80%のカバレッジ要求
   - 重要なビジネスロジックは100%

3. **テスト実行**
   ```bash
   # すべてのテストを実行
   flutter test
   
   # 特定のテストを実行
   flutter test test/path/to/test.dart
   
   # カバレッジレポート生成
   flutter test --coverage
   genhtml coverage/lcov.info -o coverage/html
   ```
