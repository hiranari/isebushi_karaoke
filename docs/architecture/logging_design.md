# ロギング設計

## 基本方針

1. インターフェース設計
- `ILogger`インターフェースはシンプルに保つ
- 基本的なログレベル（debug, info, warn, error）のメソッドのみを提供
- 特殊なユースケース用のメソッドは追加しない

2. ログ制御
- ログの有効/無効やフィルタリングはロガーの実装側で制御
- 環境変数や設定ファイル、初期化時のオプションで制御
- 呼び出し側（各サービス）はログ制御を意識しない

3. 使用例
```dart
// 良い例：シンプルな呼び出し
_logger.debug('Processing chunk: $result');

// 避けるべき例：呼び出し側でログ制御
if (_enableDebug) {
  _logger.debug('Processing chunk: $result');
}
```

## 現在の課題

1. 性能への影響
- デバッグログによる性能低下が確認されている
- 性能テストで基準値（5000ms）を超過（現在: 26332ms）
- 対応は別途検討予定（パフォーマンス最適化フェーズで対応）
